## Blocking Constraints

|----|------|-----------|

| C-003 | Never submit k8s manifests without a matching Secret template (`envsubst`-compatible) | Prevents CI deploy from failing due to missing secret injection |

| C-005 | Never introduce new crate-level dependencies without discussion | Per CONTRIBUTING.md — prevents supply chain bloat |
| C-006 | Never submit large cross-crate refactors in a single PR | Per CONTRIBUTING.md — separate library changes from service changes |
| C-007 | Never configure a Gateway without a matching Certificate resource | cert-manager must provision TLS before the gateway serves traffic |
| C-008 | Never skip the `rollout status` wait step in the deploy workflow | Silent rollout failures cause live traffic loss |

## Stack-Derived Conventions

- **Rust toolchain**: pinned to `1.86` via `rust-toolchain.toml` — do not upgrade without testing all 17 workspace members
- **Cargo workspace**: always use `-p <crate>` flags in CI/CD; never build the entire workspace in a single Docker layer
- **AT Protocol keys**: secp256k1 (k256) for JWT and repo signing; p256 for PLC rotation — do not substitute elliptic curve implementations
- **Image tagging**: always tag images with both `${{ github.sha }}` (immutable) and `latest` (rolling); k8s manifests should reference SHA tags for deterministic deploys
- **Namespace**: all ATProto services run in the `atproto` namespace (not `matrix` — that is conduit's namespace)
- **envsubst pattern**: secrets passed via environment substitution in CI, same as conduit; maintain `secret.yaml` templates with `${VAR}` placeholders
